# Plan: cvopt_participants → cv_review_requests Aktarımı

## Amaç
`cvopt_participants_rows.sql` dosyasındaki katılımcıları `cv_review_requests` tablosuna eklemek.

## Tespit Edilen Sorunlar

### 1. Tablo Hedefi Farklı
SQL dosyası `cvopt_participants` tablosuna INSERT yapıyor, ama hedef `cv_review_requests` tablosu. Yeni bir INSERT ifadesi yazılmalı.

### 2. Kolon Eşlemesi (Mapping)
| cvopt_participants | cv_review_requests | Not |
|---|---|---|
| `name` | `full_name` | Direkt |
| `linkedin` | `linkedin_url` | Direkt |
| `whatsapp` | `whatsapp_number` | Format sorunu → aşağıda |
| `approved` (boolean) | `status` (text) | `true` → `'approved'`, `false` → `'new'` |
| `linkedin_ok` (boolean) | `linkedin_reviewed` (boolean) | Direkt |
| `cv_ok` (boolean) | `cv_reviewed` (boolean) | Direkt |
| `id` (uuid) | `id` (uuid) | Aynı ID'ler korunabilir ama çakışma riski var |
| `created_at` | `created_at` | Direkt |
| `updated_at` | `updated_at` | Direkt |

### 3. WhatsApp Numara Formatı (11 kayıt sorunlu)
11 numarada format sorunu var. Bazıları ülke kodu eksik, bazıları `0` ile başlıyor:
- `905530607617` → `+905530607617` (sadece `+` eksik)
- `5458742597` → ülke kodu yok (muhtemelen `+905458742597`)
- `05511110902` → baştaki `0` kaldırılıp `+90` eklenmeli (`+5511110902`)
- `5061557989` → ülke kodu yok
- `0905433756325` → `0` kaldırılıp `+` eklenmeli
- `5308654513` → ülke kodu yok
- `015751645243` → Alman numarası (`+4915751645243`)
- `5467148137` → ülke kodu yok
- `5312825632` → ülke kodu yok
- `5382130614` → ülke kodu yok

**Not:** Bu numaralar insert edilebilir ama admin dashboard'da WhatsApp mesajı gönderilirken hatalı link üretebilir. Toplu insert'ten önce düzeltilmesi önerilir.

### 4. Duplicate Riski
Eğer bir katılımcı zaten website üzerinden CV review talebi gönderdiyse, aynı kişi iki kez listeye eklenir. `ON CONFLICT DO NOTHING` ile korunur.

### 5. UUID Çakışması
Mevcut ID'ler `cv_review_requests` tablosunda zaten varsa INSERT başarısız olur. `ON CONFLICT (id) DO NOTHING` ile çözülür. (~65 katılımcı)

## Kararlar
- **WhatsApp**: 11 sorunlu numara otomatik düzeltilecek (Türk → +90, Alman → +49, 0 temizle)
- **UUID**: Mevcut ID'ler korunacak, `ON CONFLICT (id) DO NOTHING` ile koruma

## Uygulama Planı

### Adım 1: Dönüşüm Script'i Yaz

**Kolon eşlemesi:**

### Adım 2: WhatsApp Normalize Kuralları
- `+` ile başlıyorsa → olduğu gibi
- `0` ile başlıyıp 12+ hane → `0` kaldır, `+` ekle (muhtemelen ülke kodu var)
- `0` ile başlıyıp 11 hane → Türk formatı `0` kaldır `+90` ekle
- 10 hane, `5` ile başlıyorsa → `+90` ekle
- Diğer → `+` ekle

### Adım 3: SQL Çıktısı
```sql

  ('...', 'İsim', '+905551234567', 'https://linkedin.com/...', 'approved', true, false, '...', '...'),
  ...
ON CONFLICT (id) DO NOTHING;
```
