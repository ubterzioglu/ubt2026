# GEO (Generative Engine Optimization) Improvement Plan

## Context

The homepage (`app/page.tsx`) scored 41/100 on AI search visibility. This plan addresses 15 critical issues identified in the GEO analysis. The site is a single-page portfolio for a Senior QA Engineer.

---

## Changes

### 1. Add intro summary paragraph (hero section)
**File:** `components/hero-section.tsx`
**Problem:** First paragraph is only 15 words. AI engines need 20-80 word summary for quoting.
**Fix:** Add a 2-3 sentence summary paragraph below the role text in the hero. This paragraph should:
- Be 40-60 words
- Define who UBT is in a quotable way
- Include a definition sentence pattern ("Senior QA Engineer means...")
- Mention the year 2026 for freshness signal

### 2. Convert section headings to question format (6+ question headings)
**File:** `app/page.tsx` + `components/hero-section.tsx` + `components/experience-section.tsx`
**Problem:** Only 1 question heading. Need 6+, ideally 10-15.
**Fix:** Convert existing h2 headings to question format where natural. Add sub-questions. Examples:
- "About me" → "Who Is Umut Baris Terzioglu?" (h2)
- "My CV" → "What Does UBT's CV Include?" (h2)
- "Key Achievements" → "What Are UBT's Key Achievements?" (h2)
- "Tech Stack" → "What Technologies Does UBT Use?" (h2)
- "Experience" → "What Is UBT's Professional Experience?" (h2)
- "Corporate Projects" → "Which Corporate Projects Has UBT Delivered?" (h2)
- "Private Projects" → "What Side Projects Does UBT Build?" (h2)
- "Articles" → "What Articles Has UBT Published?" (h2)
- "Book Appointment" → "How Can You Book an Appointment?" (h2)
- "CV Review" → "How Does the Free CV Review Work?" (h2)
- Add "What Is Test Automation?" or "What Is QA Engineering?" as a definition sub-heading

Update navigation items in `content/site.ts` to keep short labels but keep section IDs unchanged.

### 3. Add visible FAQ section on homepage
**File:** `app/page.tsx` (new section before footer)
**Problem:** FAQ exists only as JSON-LD (invisible). AI engines prefer visible FAQ + schema match.
**Fix:** Add a visible `<dl>/<dt>/<dd>` FAQ section (matching the geo page pattern) with the same 5 FAQ items already in the JSON-LD. Ensure visible content matches structured data exactly. Render as collapsible accordion with `<details>/<summary>` for 2+ items to satisfy the accordion requirement.

### 4. Add paragraphs throughout the page
**File:** `app/page.tsx` sections
**Problem:** Only 1 paragraph detected. Need 3-4+ standalone paragraphs.
**Fix:** Add 2-4 sentence introductory paragraphs to sections that currently only have headings + lists/cards:
- About section: already has accordion text, but add a standalone paragraph before it
- Key Achievements: add intro paragraph with stats context
- Tech Stack: add intro paragraph explaining the stack
- Corporate Projects: add intro paragraph
- Experience section: already has a paragraph, ensure it's wrapped in `<p>`
- Add a "closing" paragraph before contact section

### 5. Add definition sentences
**File:** `components/hero-section.tsx` or new content sections
**Problem:** No "X nedir?" / "What is X?" definition sentences.
**Fix:** Include definition patterns in the intro paragraph and section intros:
- "Test automation is the practice of using software tools to execute tests automatically..."
- "Quality assurance (QA) engineering is the discipline of ensuring software meets quality standards..."
- "Kısaca X, şu demektir..." pattern in English

### 6. Add a comparison table
**File:** `app/page.tsx` or new component
**Problem:** 0 tables. AI engines prefer pages with structured data tables for comparison queries.
**Fix:** Add a comparison table in one of these locations:
- Option A: In the tech stack section, compare automation tools (Selenium vs Ranorex vs Cucumber)
- Option B: In the experience section, compare roles/responsibilities across positions
- Option C: A new "QA Services Comparison" section comparing service offerings

Use semantic `<table>` with `<thead>`, `<tbody>`, `<th>`, `<td>` elements. Style with Tailwind.

### 7. Add image captions
**File:** `app/page.tsx` (corporate + private projects sections)
**Problem:** Images have no descriptive captions. AI can't understand images without text.
**Fix:** Wrap project images in `<figure>` and add `<figcaption>` elements with brief descriptions:
- Corporate project images: add caption like "Smaragd — Global PDM system used by thousands of engineers"
- Private project images: add caption like "UBT Testing — Personal QA platform for testing experience"
- Hero image: add meaningful alt text instead of empty string

### 8. Add concrete statistics/numbers in body text
**File:** `content/profile.ts` + `app/page.tsx`
**Problem:** 0 statistics detected in body text despite having achievement metrics in cards.
**Fix:** Weave concrete numbers into paragraph text:
- "15+ years of professional experience" (already in some places, ensure in paragraphs)
- "Raised test coverage from 50% to 90%"
- "Optimized test execution time by 40%"
- "30,000+ users served"
- "5,000+ enterprise test cases managed"
- Ensure these appear in `<p>` elements, not just card values

### 9. Add current year reference
**File:** `app/page.tsx` or `components/hero-section.tsx`
**Problem:** No 2026 reference in body text. Footer has ©2026 but that's not in body content.
**Fix:** Include "2026" in the intro summary paragraph and/or section intros. E.g., "As of 2026, Umut brings 15+ years of experience..."

### 10. Add publish date and update date
**File:** `app/page.tsx`
**Problem:** No visible `<time>` elements. AI engines can't determine content freshness.
**Fix:** Add near the top of the page (after hero or as part of about section):
- Published date: `<time datetime="2026-04-06">April 6, 2026</time>`
- Last updated: `<time datetime="2026-04-12">April 12, 2026</time>`
- Use `datePublished` and `dateModified` in the Person/WebSite schema

### 11. Add Speakable schema
**File:** `lib/structured-data.ts` + `app/page.tsx` or `app/layout.tsx`
**Problem:** No Speakable schema for voice assistants (Google Assistant, Siri).
**Fix:** Add `buildSpeakableSchema()` function to structured-data.ts:
```typescript
export function buildSpeakableSchema() {
  return {
    "@context": "https://schema.org",
    "@type": "SpeakableSpecification",
    "xpath": [
      "/html/head/title",
      "/html/body/main/section[@id='about-me']//p",
      "/html/body/main/section[@id='key-achievements']//p"
    ],
    "cssSelector": [
      "#about-me p",
      "#key-achievements p"
    ]
  };
}
```
Add to homepage JSON-LD injection.

### 12. Add internal links to geo pages
**File:** `app/page.tsx`
**Problem:** No topical internal links. The geo pages exist but aren't linked from the homepage.
**Fix:** Add contextual text links to geo pages from relevant sections:
- From about/contact: "Available for QA consulting in [Dortmund](/geo/dortmund), [Frankfurt](/geo/frankfurt), [Berlin](/geo/berlin), and other German cities."
- From experience section: link to location-relevant geo pages
- Use Next.js `<Link>` component for internal navigation

### 13. Add expertise/originality signals
**File:** `app/page.tsx` or new content section
**Problem:** Content reads as general description; no unique methodology or case study.
**Fix:** Add a brief methodology/approach section or weave expertise signals into existing content:
- "UBT's testing methodology combines risk-based testing with shift-left principles..."
- Reference specific frameworks or proprietary approaches
- Add a "How I Work" or "My Approach" mini-section
- Include first-person voice in about section for authenticity

### 14. Improve render/crawl quality
**File:** `app/layout.tsx` + `app/page.tsx`
**Problem:** Content-to-HTML ratio may be low due to inline scripts and heavy components.
**Fix:** 
- Move analytics scripts to `strategy="lazyOnload"` instead of `afterInteractive` (they don't affect content)
- Ensure main content is in initial HTML (it already is since these are server components)
- This is a minor concern; the actual fix may just be improving content density via the other changes

### 15. Cover alt intent clusters
**File:** `app/page.tsx` headings + content
**Problem:** 0/6 intent clusters covered. Need to address: price, how-to, comparison, problem/solution, definition.
**Fix:** Through the heading changes and new content, ensure these intents are covered:
- **Definition intent:** "What Is QA Engineering?" heading/paragraph
- **How-to intent:** "How Does the Free CV Review Work?" heading + steps
- **Comparison intent:** Tech stack comparison table
- **Problem/solution intent:** "Common QA Challenges and How I Solve Them" paragraph or heading
- **Price intent:** "Free CV review" and "free appointment booking" mentioned clearly
- **Location intent:** Already covered via geo pages, link them from homepage
